Vanilla Databinding

I found this function here in an attempt to understand how databinding really works before jumping into a framework or library like svelte or react.

This is ment to be a preliminary excercise before diving deeper into the features and methods various framework authors approach the subject.

                function HTMLBinding(b) {

                    this.elementBindings = [];
                    this.value = b.object[b.property]
                
                    this.valueGetter = () => {
                        return this.value;
                    }
                
                    this.valueSetter = (val) => {
                        this.value = val;
                        this.elementBindings
                            .forEach(binding => binding[binding.attribute] = val);
                    }        
                
                    this.addBinding = (element,attribute,event) => {
                        const binding = {
                            element,
                            attribute,
                        }
                
                        if (event)
                            element.addEventListener(
                                event, 
                                () => this.valueSetter(element[attribute]))
                        
                        binding.event = event;
                        this.elementBindings.push(binding)
                        element[attribute] = this.value;
                        return this;
                    }
                
                    Object.defineProperty(b.object,b.property, {
                        get: this.valueGetter,
                        set: this.valueSetter
                    });
                
                    b.object[b.property] = this.value;
                
                }

The Exercises

  • Input Fields

    1. Create an form that binds its value to a user bio element
    2. Then a button that on save grabs that form data, using it to interact with and update multiple elements
    3. Bind the button state [enabled/disable] to validation rules within the form

  • Product Templates

    1. Create three separate product representations syncing the following data across each one
      1. Image
      2. Price
      3. Description
      4. Name
    2. Create an admin like feature panel/pop-up that allows you to update the name, description, and image across all of them

  • Tab Handler

    1. Create a ui component that displays and updates the following based on which tab is clicked
      1. Tab Name
      2. Sub Links
